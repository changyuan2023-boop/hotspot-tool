# 热点监控 · 线上部署说明

将 `hotspot_monitor_api.py` + `hotspot_monitor.html` 部署到一台有公网 IP 的服务器后，可通过 `https://你的域名/` 或 `http://服务器IP:端口/` 访问，无需再在本机跑后端。

---

## 一、前置条件

- 一台**有公网 IP** 的 Linux 服务器（阿里云 / 腾讯云 / 其他 VPS 均可）。
- 服务器已安装 **Python 3.8+**。
- 本地的 **LongPort 凭证**（`.env` 里的三项）准备在服务器上配置，**不要**把 `.env` 提交到 git。

---

## 二、部署步骤（通用 Linux 服务器）

### 1. 上传代码

将以下文件放到服务器同一目录（例如 `/opt/hotspot-monitor/`）：

- `hotspot_monitor_api.py`
- `hotspot_monitor.html`
- `requirements.txt`

（不要上传 `.env`，在服务器上单独创建。）

### 2. 在服务器上创建虚拟环境并安装依赖

```bash
cd /opt/hotspot-monitor   # 换成你的目录
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3. 配置环境变量（LongPort 凭证）

在服务器上新建 `.env` 文件（与 `hotspot_monitor_api.py` 同目录）：

```bash
nano .env
```

填入（与本地一致）：

```
LONGPORT_APP_KEY=你的AppKey
LONGPORT_APP_SECRET=你的AppSecret
LONGPORT_ACCESS_TOKEN=你的AccessToken
```

保存后设置权限，避免被其他用户读取：

```bash
chmod 600 .env
```

### 4. 用 Gunicorn 启动（生产环境）

开发时用 `python3 hotspot_monitor_api.py`，线上建议用 **Gunicorn**：

```bash
source venv/bin/activate
gunicorn -w 2 -b 0.0.0.0:5000 --chdir /opt/hotspot-monitor hotspot_monitor_api:app
```

- `-w 2`：2 个 worker，可按 CPU 核数调整。
- `-b 0.0.0.0:5000`：监听所有网卡的 5000 端口，外网可访问。
- 若需改端口，把 `5000` 改成其他端口即可。

此时在浏览器访问 **http://服务器公网IP:5000/** 即可打开热点监控页。

### 5. 保持常驻（systemd，推荐）

避免关掉终端后进程退出，可用 systemd 托管：

```bash
sudo nano /etc/systemd/system/hotspot-monitor.service
```

写入（路径按你实际目录改）：

```ini
[Unit]
Description=Hotspot Monitor API
After=network.target

[Service]
Type=simple
User=你的用户名
WorkingDirectory=/opt/hotspot-monitor
Environment="PATH=/opt/hotspot-monitor/venv/bin"
ExecStart=/opt/hotspot-monitor/venv/bin/gunicorn -w 2 -b 0.0.0.0:5000 hotspot_monitor_api:app
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

然后：

```bash
sudo systemctl daemon-reload
sudo systemctl enable hotspot-monitor
sudo systemctl start hotspot-monitor
sudo systemctl status hotspot-monitor
```

### 6. 防火墙放行端口

若云厂商有安全组/防火墙，需放行你使用的端口（如 5000）：

- 阿里云：安全组入方向允许 TCP 5000。
- 腾讯云：防火墙入站规则允许 5000。
- 本机若开了 firewalld：`sudo firewall-cmd --add-port=5000/tcp --permanent && sudo firewall-cmd --reload`。

---

## 三、可选：Nginx 反代 + HTTPS

若希望用域名并上 HTTPS（推荐）：

1. 安装 Nginx，申请域名并解析到服务器 IP。
2. 用 Let's Encrypt 申请证书（如 `certbot`）。
3. Nginx 配置示例：

```nginx
server {
    listen 80;
    server_name 你的域名;
    return 301 https://$server_name$request_uri;
}
server {
    listen 443 ssl;
    server_name 你的域名;
    ssl_certificate     /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;
    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Gunicorn 可改为只监听本机：`-b 127.0.0.1:5000`，由 Nginx 对外提供 443。

---

## 四、前端说明

- 页面由后端在根路径 `/` 提供，前端请求 `/api/quotes` 时使用当前页面的 `origin`，因此部署到任意域名或 IP 后**无需改前端代码**，打开对应地址即可使用。

---

## 五、安全检查

- 不要将 `.env` 或含凭证的文件提交到 git 或上传到公开位置。
- 若仅内网使用，可在安全组/防火墙中限制仅允许指定 IP 访问 5000 或 443。
- 生产环境务必用 HTTPS（Nginx + 证书），避免凭证或数据在传输中被窃听。

---

## 六、小结

| 项目       | 说明 |
|------------|------|
| 运行命令   | `gunicorn -w 2 -b 0.0.0.0:5000 hotspot_monitor_api:app`（在项目目录下，或 `--chdir` 指定目录） |
| 访问地址   | `http://服务器IP:5000/` 或 `https://你的域名/`（若配置了 Nginx） |
| 环境变量   | 服务器上同目录 `.env` 中配置 LongPort 三项，与本地一致 |
| 常驻方式   | systemd 或 screen/tmux，推荐 systemd |

按上述步骤即可将热点监控整体运行在线上。
