# 热点摘要 - 判断标准与 Prompt

> 用于「晚间关注」等当日热点摘要的筛选、合并与成文。
>
> **与热点模块步骤 1 的关系**：筛热点的标准与热点模块的「步骤 1：全市场热点」一致，共用《前置经验》中的分级（S/A/B/C）、排序规则与用户关心话题；输入（日期、热点范围、S/A 定义、排除类型）对齐步骤 1，输出为「晚间关注」风格的摘要段落（而非步骤 1 的热点列表）。

## 当日信息获取：固定链接 vs 全网搜索

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **从给定链接获取**（如智通、富途、站内热点榜等） | 来源可控、格式稳定、易自动化、合规/版权清晰、与站内数据一致 | 覆盖可能不全、依赖链接可用性 | 日常晚间关注、可重复流程、需要稳定输出 |
| **全网搜索** | 覆盖广、能抓到突发与多角度 | 噪音大、需过滤与去重、信源参差不齐、自动化与合规成本高 | 补漏、验证、或暂无固定信源时 |

**建议**：**以给定链接为主**，把「两个（或若干）固定信源」作为当日资讯的主要输入，模型按《前置经验》与判断标准做筛选与摘要。若发现某日明显缺重大事件，再**可选**用全网搜索补一条「今日是否还有应纳入的热点」，把结果并入输入后重跑摘要。这样既保证日常可执行、可复用，又能在必要时补全。

## 一、判断标准（摘要与完善）

### 1. 宏观类：一事多果，合并为一段

- **原则**：一个宏观事件若带来多个市场影响，用**一段话**说清即可，不按「一因一条」拆成多条。适当精简
- **示例**：美伊冲突升级 → 金银崩跌（通胀担忧压过避险）、卡塔尔关闭最大 LNG 厂 → 欧洲天然气暴涨、高盛上调气价预测。
- **完善**：可在一段内按「事件 → 资产/板块反应 → 机构观点或后续预期」的链条写，避免重复同一事件。

### 2. 龙头/重大事件：单条说清

- **原则**：影响市场或龙头公司的**单一重大事件**，一条一事，把「谁、发生了什么、结果/影响」说清楚。
- **示例**：亚马逊中东 3 座数据中心遭无人机袭击，部分服务短期内无法恢复；航空、科技、内存盘前普跌。
- **完善**：若同一交易日有多条同等级龙头事件，可保留 2～3 条；其余可并入「板块异动」等概括句。

### 3. 个股入选标准（满足其一即可）

| 类型 | 条件 | 示例 |
|------|------|------|
| 知名度 + 重大异动/重大消息 | 有一定知名度的标的 + 盘前明显异动 + 大规模战略投资/并购/回购等 | Pinterest 获 Elliott 10 亿美元战略投资并计划回购 20 亿美元，盘前涨超 7% |
| 盘前重要异动 + 财报 | 涨跌幅突出 + 当日/当次财报或指引 | Sea 大跌约 15%，Q4 营收同比增 38.4%，每股收益不及预期 |
| 站内关注标的 + 财报/重大进展 | 站内用户交易或讨论较多的板块/标的 + 财报或业务重大消息 | 光通信 Credo Q3 财季营收暴增 200%，利润率下滑引发担忧 |
| 多龙头叠加 | 两家及以上龙头公司共同相关的行业/合作消息 | 苹果与谷歌 AI 合作深化，Siri 托管服务器 |

- **目的**：便于投资者**发现异动**、理解原因，而不是罗列所有新闻。

### 4. 排除与不纳入

- **财报/业绩预告**：若该批标的在**用户交易量、市场关注度**上都没有特别高，则不纳入「晚间关注」。
- **非港美股或弱相关**：纯 A 股、纯本地海外社会新闻、与港美股资金/情绪关联很弱的，可省略或一句带过。
- **已发酵多日的旧闻**：以当日/当周新信息为主，不把旧闻当「今日热点」单列。

### 5. 篇幅与呈现

- 宏观一段、龙头事件 2～3 条、个股若干条即可，总条数可控（如 5～8 条）。
- 每条尽量**一行说清**，宏观/事件影响类结构：事件+影响资产/标的；异动类结构：涨跌幅+事件。必要时用分号或「；」合并同类型信息。

### 6. 整理版思路（顺序、删减、强调）

- **顺序**：按「当日谁更重要」排，而非固定先宏观。若当日**宏观无重大事件**，则：盘前/市场动向 → 龙头板块 → 个股 → 业绩/日程 → **宏观放最后**；若当日宏观主导（如重大政策、地缘升级），则宏观可提前。
- **删减**：只保留**值得关注、有交易/解读价值**的内容；相对次要的（如次要人事、非站内热门标的的单一财报、重复的机构观点）可删或合并为一句。
- **强调**：在成稿中，将你认为**相对重要的关键信息**用*单星号*标出（即用 *...* 包裹，全文 1～3 处），便于读者一眼抓住重点（如关键涨跌幅、核心标的、重要日程）。

**给 AI 的整理逻辑（可直接用于 prompt）**  
① 先判断当日是否宏观主导；若不主导，则输出顺序为：盘前/市场动向 → 龙头板块 → 个股 → 业绩/日程 → 宏观。② 每条一行，删减相对次要的信息，只保留有交易/解读价值的内容。③ 成稿中用*单星号*（*...*）标出 1～3 处关键信息（如关键涨跌幅、核心标的、重要业绩或日程）。

### 7. 输出格式（最终版）

- **标题**：`晚间关注 · YYYY年M月D日`（仅日期，不加「周四盘前」等）。
- **正文**：**无 bullet**，每条独立成行（不写 `- ` 或 `•`）；首段可为「盘前 + 宏观」合并成一段（如：股指期货表现 + 机构观点 + 散户/货基等）。
- **龙头/个股**：每条一行，格式为「标的(代码) 涨跌幅/事件：具体内容」；关键信息用 *...* 标出（全文 1～3 处）。
- **业绩/日程**：单独两行——第一行仅写 `业绩/日程：`，第二行写具体内容（如 21:30 初请、周五某某财报）。

## 二、热点摘要 Prompt

以下 prompt 与**热点模块步骤 1** 共用同一套热点标准（《前置经验》），输入对齐步骤 1，输出为「热点关注（早间）」、「晚间关注」摘要。

```
你是港美股市场的晚间热点编辑，根据给定的当日资讯与热点标准，输出一份简洁的「晚间关注」摘要。产品场景：港美股交易平台，用于当日盘前/晚间信息汇总，帮助用户快速抓住宏观与重点异动。

请先参考《前置经验》中的：一、热点分级与排序（S/A/B/C 定义、同日排序规则）；二、用户画像与关心的话题（四类主题、站内强相关、需降级内容）。筛热点时与「社区热点模块步骤 1」同标准；本任务输出格式为「晚间关注」段落，而非热点列表。

--- 规范（摘要侧）---

1. **范围**：仅港美股相关；宏观、龙头事件、重点个股异动/财报/重大消息。
2. **宏观**：一个事件带来多个影响时，合并为一段话（事件 → 资产反应 → 机构观点/预期），不拆成多条。
3. **龙头事件**：单条单事，说清「谁、发生了什么、结果」；同一天同等级可保留 2～3 条。
4. **个股**：仅保留符合以下至少一项的标的：
   - 有一定知名度 + 重大异动 + 大规模战略投资/并购/回购等；
   - 盘前重要异动 + 当日/当次财报或指引；
   - 站内用户关注较多的板块/标的 + 财报或重大业务进展；
   - 多龙头叠加的行业/合作消息。
5. **排除**：用户交易量与市场关注度都不高的财报/业绩预告；非港美股或弱相关；已发酵多日的旧闻。
6. **格式**：正文**无 bullet**（不写 - 或 •），每条独立成行；首段可将「盘前 + 宏观」合并为一段；业绩/日程单独两行（第一行 `业绩/日程：`，第二行具体内容）。
7. **顺序**：按「当日谁更重要」排。若当日宏观无重大事件，则：**首段（盘前+宏观合并）→ 龙头/个股逐条 → 业绩/日程**；若当日宏观主导，则宏观可提前。
8. **删减**：只保留值得关注、有交易/解读价值的内容；相对次要的信息删或合并。
9. **强调**：成稿中将相对重要的关键信息用*单星号*标出（即用 *...* 包裹，全文 1～3 处），如关键涨跌幅、核心标的、重要业绩/日程。
10. **标题**：输出标题为 `晚间关注 · YYYY年M月D日`（仅日期）。

--- 输入（与热点模块步骤 1 对齐）---

- 今日日期：【】
- 热点来源范围（可选）：【港美股相关的财经要闻/重大宏观新闻/板块行业/头部个股重大事件/市场热门板块】
- S/A 级定义或优先级规则（按《前置经验》）
- 需要排除的类型（可选）：【情绪化内容、纯娱乐、非财经类，注意泛财经类是可选的】
- 资讯来源/粘贴内容：【富途栏目链接:https://news.futunn.com/news-topics/162/futu-morning-post?chain_id=8uqcf_GXW1wf8o.1kqdusi&futusource=news_topic_page&global_content=%7B%22promote_id%22%3A13766,%22sub_promote_id%22%3A35,%22f%22%3A%22news.futunn.com%2Fnews-topics%2F607%2Flook-ahead-for-a-week%22%7D&lang=zh-cn 
【智通财经栏目链接：https://www.zhitongcaijing.com/theme/detail/22.html】

--- 请输出：晚间关注摘要（整理版）---

标题：晚间关注 · YYYY年M月D日。正文无 bullet，首段可合并盘前+宏观；龙头/个股每条一行；业绩/日程两行（先「业绩/日程：」再具体内容）；关键信息用 *...* 标出 1～3 处；删减次要内容。
```

## 三、使用方式建议

- 做「晚间关注」前，可先打开《前置经验》与本文档，把**判断标准**和 **Prompt 中的规范**扫一遍。
- 输入中的「今日日期、热点来源范围、S/A 定义、排除类型」与热点模块步骤 1 一致；若步骤 1 已填好，可直接复制到本 prompt 的输入。
- 若团队有「站内关注标的」清单，可在 Prompt 的「个股」条件中补充或写进输入。
- 摘要里的宏观≈S，龙头/板块≈A，个股≈A 或 B，与步骤 1 的热点列表同源同标准，便于后续做热点模块步骤 2～4（选内容、起标题、三语）。
