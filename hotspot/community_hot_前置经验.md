# 热点运营 - 前置经验（知识库）

> 本文档沉淀热点模块所需的前置经验。使用 prompt 前可先阅读或引用本文；待本文填满后，可在步骤 1～4 的 prompt 中增加「请参考《前置经验》」或直接粘贴相关段落。

## 一、热点分级与排序

### 1.1 分级定义（S / A / B / C）

| 等级 | 定义 | 典型例子 |
|------|------|----------|
| **S 级** | 宏观新闻类 / 热门个股 Top5 | 中美贸易战、英伟达财报、宏观政策发布 |
| **A 级** | 火热概念相关，影响多支股票，热度有持续性，对市场有明显影响的事件 | 量子计算、AI 热潮、美联储议息结果、联合健康 |
| **B 级** | 二、三线个股异动、热门概念股的单支股票消息 | 文远知行、AAPL 单日异动、阿里财报 |
| **C 级** | 冷门股或迷你股突发爆量，有炒作现象 | 低价股、Meme 股的暴涨 |

### 1.2 同一天多个热点如何排序？

- **优先级规则**：有 S 级热点优先；若无 S 级，则「热门板块异动（多股联动）」=「龙头财报」>「热门交易」。
- **同等级时的取舍**：站内是否有优质内容、用户持仓/交易相关度。
- **每日模块数量**：【请填写：例：通常 1 条，大日可 2 条】

### 1.3 不同类型热点的排序经验

| 类型 | 热点模块典型标题 | 通常优先级 | 备注 |
|------|------------------|------------|------|
| 宏观/政策 | 伊朗冲突持续，该如何操作？ | S | 宏观动荡，影响重大，优先 |
| 龙头异动 | 英伟达绩后下跌，你怎么看？ | A | 龙头股叠加财报，优先 |
| 财报/业绩 | 英伟达财报超预期！如何解读？ | A | 财报发布叠加龙头股，重要 |
| 行业/板块 | AI「双雄」智谱、MINIMAX 大跌，是吸纳良机？ | A | AI 板块，多标的联动，重要 |
| 个股重大 | 泡泡玛特六日连涨，如何解读？ | B | 多日异动，当日无重点热点时可补位 |
| 其他 | SpaceX 宣布收购 xAI，如何解读？ | B | 特斯拉沾边，投资用户喜欢 |

## 二、用户画像与关心的话题

> 用于判断「什么事件值得做热点」、用户喜欢的角度是什么，以及步骤 1 筛热点时的参考。

### 2.1 用户画像

- **身份与行为**：港美股投资者/交易者，关心头部科技股，投资风格多样；会根据新闻与市场热点寻找投资机会。
- **关注与偏好**：关注特大型宏观事件（贸易/地缘）、龙头财报与指引（如 MAG7、小米集团）、火热概念（AI、光通信、存储芯片、量子）、二三级个股异动、站内热门交易/涨跌；偏好有逻辑的解读与可操作角度（如「该买哪些标的」「某事件如何解读」「财报前如何买期权」「宏观下选哪些避险标的」），标题与内容要呼应、一眼能懂；对美股 MAG7、港股小米与中概等兴趣高；对泛财经也很感兴趣，如马斯克博客采访稿、巴菲特股东信、投资心法等，是一群爱学习的投资者
- **内容与合规期待**：希望及时、准确、中性；不喜煽动与荐股口吻，但接受有观点的讨论与解读。
- **触达与语言**：触达场景包括 App 内 Feed、推送、要闻置顶/头图、个股资讯/讨论置顶、AI 气泡、策划活动等；语言覆盖简中、繁中、英文，三语标题/文案需一致、地道。

### 2.2 用户常关心的主题分类（四类）

| 类型 | 特点 | 运营要点 | 举例 |
|------|------|----------|------|
| **第一类：重点异动** | 最高频的每日滚动热点，需每日多次维护 | 持续跟踪、及时选稿与置顶 | 恒强标的：美股 MAG7、港股小米/腾讯、中概股；主线轮动：AI 主线 Minimax/智谱，新消费泡泡玛特/蜜雪冰城，存储芯片闪迪/美光，避险大宗白银/黄金；用户常买 ETF：大盘 TQQQ、半导体 SOXL、特斯拉二倍 TSLL 等 |
| **第二类：财报相关** | 季度轮动，头部公司效应强，中尾部公司在财报前后也有异动 | 财报前中后全链路内容 | 龙头财报、中尾部财报异动引发关注 |
| **第三类：宏观数据发布** | 月度、周度事件 | 按节奏配置要闻/头图/解读 | 非农、CPI、美联储议息等 |
| **第四类：行业板块轮动** | 持续时间较长或间歇有消息 | 除事实性内容外，需多角度解读与主题股单；链路：「知道事件 → 了解事件 → 清楚影响与趋势 → 买卖标的 → 操作策略」 | 例：黄金长期上涨时，除股票外可做金银 ETF 标的选择等运营 |

### 2.3 与「站内交易/持仓」强相关的话题（优先做运营）

- 实时热点票来源：【从此处获取 https://algo-tianwen.longbridge-inc.com/demo/hot-stock-rank】

### 2.4 用户不太关心或需降级的内容

- 过小市值、纯海外本地新闻、已发酵多日的旧闻。
- **例外**：部分公司知名度一般但仍有跟踪价值，例如 Pinterest（社媒）、Shopify（独立站/社媒影响力）、Palantir（用户喜爱）、Credo（光通信概念）。

## 三、好的热点内容标准（资讯 vs 社区）

> 用于选稿、置顶、推送时判断：单条内容是否适合作为热点载体。按「资讯」与「社区」两类分别定义，便于步骤 2（站内内容匹配）及各类运营动作统一标准。

### 3.1 资讯类「好内容」标准

| 维度 | 标准说明（请按团队共识填写） |
|------|------------------------------|
| 来源/权威性 | 【例：权威媒体、机构研报、官方披露；禁止无来源或营销号】 |
| 时效性 | 【例：当日/当周；与热点发生时间匹配】 |
| 信息密度与结构 | 【例：事实清晰、有数据或信源、结构完整；禁止标题党、空洞解读】 |
| 与热点的相关度 | 【例：直接解读该热点或核心标的，非泛泛提及】 |
| 禁止/排除 | 【例：软广、荐股、煽动、抄袭、已过期旧闻】 |

### 3.2 社区类「好内容」标准

| 维度 | 标准说明（请按团队共识填写） |
|------|------------------------------|
| 作者/互动 | 【例：真实用户、有一定互动或历史内容质量；禁止水军、营销号】 |
| 观点与逻辑 | 【例：有明确观点或可操作角度、论述有逻辑；禁止纯情绪、无信息量】 |
| 与热点的相关度 | 【例：紧扣该热点或相关标的，可引发讨论】 |
| 形式与可读性 | 【例：标题与正文一致、易读；禁止标题党、纯搬运无点评】 |
| 禁止/排除 | 【例：软广、荐股、人身攻击、违规内容】 |

### 3.3 两类共用原则

- 合规：语言中立，不荐股不煽动，符合平台与金融信息规范。
- 与「热点分级」匹配：S/A 级热点优先选资讯类权威解读或社区高互动优质讨论；B/C 级可放宽至优质社区帖。

## 四、推荐-社区热点模块：常见标题与格式

> 用于步骤 3（话题命名）和步骤 4（三语标题）时参考，保证风格一致、与帖子内容呼应、用户一眼能懂。

### 4.1 横幅标题的聚焦维度：公司、股票优先

- 热点模块横幅标题需**适当聚焦在公司、股票维度**，而非产品发布或产品细节。
- **宜强调**：公司战略、市场反应、估值/投资逻辑、业务与营收影响、投资者关切（如「对 AAPL 意味着什么」「市场怎么看」）。
- **避免**：仅复述产品发布、型号名称、芯片/参数等产品细节；若热点由某款产品触发，话题名称应把用户视线引向「对这家公司/这只股票意味着什么」「如何解读」，而不是「某产品发布了」「某配置是什么」。
- **思路**：从产品/事件切入时，标题落到公司层面（如「苹果 A 芯片战略」「市场怎么看苹果布局」），便于投资与交易场景下的用户一眼理解与点击。

| 避免（偏产品/细节） | 推荐（聚焦公司/股票） |
|--------------------|------------------------|
| MacBook Neo 用 A18 Pro，为「手机变 Mac」铺路？ | 苹果 A 芯片战略：清库存还是为「手机即电脑」铺路？ |
| 从 MacBook Neo 看苹果「一机两用」可能 | 苹果「一机两用」猜想：对业务与股价意味着什么？ |
| 苹果新 Mac 搭手机芯片，你怎么看？ | AAPL：从新 Mac 看公司芯片与产品线战略 |

### 4.2 常用标题结构/句式

- 「贵金属牛市来袭？该怎么买？」
- 「白宮《國家安全戰略》如何解讀？」
- 「泡泡瑪特大跌，你的持倉還好嗎？」
- 「Gold Miners vs ETFs: Which to Buy?」
- 「Trump 2.0's New Policies: US Stocks Rally Again!」

### 4.3 避免的标题风格

- 【请填写：例：纯疑问无信息、夸张/煽动、荐股口吻、过长、冷门缩写】

### 4.4 字数与格式（硬性要求）

- **简中/繁中**：横幅话题名**必须**在 **15 字符以内**（含标点）。超出即不合格。
- **英文**：与中文语义一致，翻译准确，写成地道英文社区话题标题，尽量简短。

### 4.5 生成热点模块标题：默认输出规则

- **默认一次性输出「一组三语标题」**：每条标题都给出简体中文、繁体中文、英文，三语语义一致。
- **若有多个标题**（如主推 1～3 条 + 备选）：则输出**多组**，每组包含「简中 + 繁中 + 英文」三条，组与组对应同一条标题的不同语言版本。
- **每组内简中/繁中必须 ≤15 字**（硬性标准）；英文保持简短、标题风格。
- 示例：主推 2 条 + 备选 1 条 → 输出 3 组，每组 3 行（简中 / 繁中 / 英文）。

## 六、站内内容 ID 获取方法

> 从 LongBridge 站内链接中解析内容 ID，便于步骤 2（站内内容匹配）、配置热点模块载体时使用。

| 类型 | 链接示例 | ID 位置 | 示例 ID |
|------|----------|---------|---------|
| **帖子（Topics）** | `https://longbridge.com/zh-CN/topics/39055244?channel=WHAB0002` | 链接中 **topics/** 后的数字 | 39055244 |
| **资讯（News）** | `https://longbridge.com/zh-CN/news/277749821?channel=WHAB0002&invite-code=563443` | 链接中 **news/** 后的数字 | 277749821 |

**规则**：取 URL 路径里 `topics/` 或 `news/` 后面、到下一个 `/` 或 `?` 之前的一串数字，即为该条内容的内容 ID。

## 七、其他约定

- **时效**：【请填写：例：当日/当周热点才做模块】
- **合规/禁用**：语言中立、合规，不荐股不煽动。
- **与竞品/行业的差异**：热点模块相对活泼；社区偏去中心化，更多在内容分发上做「不留痕迹」的动作，较少主动设置议程、强行向用户分发热点。

## 使用方式建议

1. 先按团队共识把文中【】填满，形成稳定版本。
2. **步骤 1（筛热点）**：在 prompt 中增加「请参考《前置经验》一、二：热点分级、排序与用户关心话题」。
3. **步骤 2（选内容）**：在 prompt 中增加「请参考《前置经验》三：好的热点内容标准（资讯 vs 社区）」。
4. 需要时把「常见标题句式」「用户关心话题」「好内容标准」等段落直接复制进对应步骤的 prompt，减少模型自由发挥、提高一致性。
