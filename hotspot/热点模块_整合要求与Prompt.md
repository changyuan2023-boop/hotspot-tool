# 热点模块 · 整合要求与 Prompt

> 本文档整合《前置经验》与《社区-热点模块 · Prompt》中的全部要求，便于一次性查阅或交给大模型使用。可按步骤 1→2→3→4 顺序执行，也可单独使用某一步（如仅用步骤 3+4 起标题）。

---

## 一、热点分级与排序

### 1.1 分级定义（S / A / B / C）

| 等级 | 定义 | 典型例子 |
|------|------|----------|
| **S 级** | 宏观新闻类 / 热门个股 Top5 | 中美贸易战、英伟达财报、宏观政策发布 |
| **A 级** | 火热概念相关，影响多支股票，热度有持续性，对市场有明显影响的事件 | 量子计算、AI 热潮、美联储议息结果、联合健康 |
| **B 级** | 二、三线个股异动、热门概念股的单支股票消息 | 文远知行、AAPL 单日异动、阿里财报 |
| **C 级** | 冷门股或迷你股突发爆量，有炒作现象 | 低价股、Meme 股的暴涨 |

### 1.2 同一天多个热点如何排序

- **优先级规则**：有 S 级热点优先；若无 S 级，则「热门板块异动（多股联动）」=「龙头财报」>「热门交易」。
- **同等级时的取舍**：站内是否有优质内容、用户持仓/交易相关度。
- **每日模块数量**：【请填写：例：通常 1 条，大日可 2 条】

### 1.3 不同类型热点的排序经验

| 类型 | 热点模块典型标题 | 通常优先级 | 备注 |
|------|------------------|------------|------|
| 宏观/政策 | 伊朗冲突持续，该如何操作？ | S | 宏观动荡，影响重大，优先 |
| 龙头异动 | 英伟达绩后下跌，你怎么看？ | A | 龙头股叠加财报，优先 |
| 财报/业绩 | 英伟达财报超预期！如何解读？ | A | 财报发布叠加龙头股，重要 |
| 行业/板块 | AI「双雄」智谱、MINIMAX 大跌，是吸纳良机？ | A | AI 板块，多标的联动，重要 |
| 个股重大 | 泡泡玛特六日连涨，如何解读？ | B | 多日异动，当日无重点热点时可补位 |
| 其他 | SpaceX 宣布收购 xAI，如何解读？ | B | 特斯拉沾边，投资用户喜欢 |

## 二、用户画像与关心的话题

### 2.1 用户画像

- **身份与行为**：港美股投资者/交易者，关心头部科技股，投资风格多样；会根据新闻与市场热点寻找投资机会。
- **关注与偏好**：关注特大型宏观事件（贸易/地缘）、龙头财报与指引（如 MAG7、小米集团）、火热概念（AI、光通信、存储芯片、量子）、二三级个股异动、站内热门交易/涨跌；偏好有逻辑的解读与可操作角度（如「该买哪些标的」「某事件如何解读」「财报前如何买期权」「宏观下选哪些避险标的」），标题与内容要呼应、一眼能懂；对美股 MAG7、港股小米与中概等兴趣高；对泛财经也很感兴趣。
- **内容与合规期待**：希望及时、准确、中性；不喜煽动与荐股口吻，但接受有观点的讨论与解读。
- **触达与语言**：触达场景包括 App 内 Feed、推送、要闻置顶/头图、个股资讯/讨论置顶等；语言覆盖简中、繁中、英文，三语标题/文案需一致、地道。

### 2.2 用户常关心的主题分类（四类）

| 类型 | 特点 | 运营要点 |
|------|------|----------|
| **第一类：重点异动** | 最高频的每日滚动热点 | 持续跟踪、及时选稿与置顶；恒强标的：美股 MAG7、港股小米/腾讯、中概股；主线轮动：AI 主线、新消费、存储芯片、避险大宗等 |
| **第二类：财报相关** | 季度轮动，头部公司效应强 | 财报前中后全链路内容 |
| **第三类：宏观数据发布** | 月度、周度事件 | 按节奏配置要闻/头图/解读（非农、CPI、美联储议息等） |
| **第四类：行业板块轮动** | 持续时间较长或间歇有消息 | 多角度解读与主题股单 |

### 2.3 站内强相关与需降级

- **优先做运营**：与站内交易/持仓强相关的话题；实时热点票来源可参考站内热股榜。
- **需降级或不纳入**：过小市值、纯海外本地新闻、已发酵多日的旧闻。例外：部分公司知名度一般但仍有跟踪价值（如 Pinterest、Shopify、Palantir、Credo）。

## 三、好的热点内容标准（资讯 vs 社区）

用于步骤 2（站内内容匹配）及选稿、置顶时统一标准。

### 3.1 资讯类「好内容」

- 来源/权威性：权威媒体、机构研报、官方披露；禁止无来源或营销号。
- 时效性：当日/当周；与热点发生时间匹配。
- 信息密度与结构：事实清晰、有数据或信源、结构完整；禁止标题党、空洞解读。
- 与热点的相关度：直接解读该热点或核心标的，非泛泛提及。
- 禁止/排除：软广、荐股、煽动、抄袭、已过期旧闻。

### 3.2 社区类「好内容」

- 作者/互动：真实用户、有一定互动或历史内容质量；禁止水军、营销号。
- 观点与逻辑：有明确观点或可操作角度、论述有逻辑；禁止纯情绪、无信息量。
- 与热点的相关度：紧扣该热点或相关标的，可引发讨论。
- 形式与可读性：标题与正文一致、易读；禁止标题党、纯搬运无点评。
- 禁止/排除：软广、荐股、人身攻击、违规内容。

### 3.3 共用原则

- 合规：语言中立，不荐股不煽动，符合平台与金融信息规范。
- 与热点分级匹配：S/A 级热点优先选资讯类权威解读或社区高互动优质讨论；B/C 级可放宽至优质社区帖。

## 四、热点模块标题的聚焦与格式（起标题必读）

### 4.1 横幅标题：公司、股票维度优先

- 热点模块横幅标题需**适当聚焦在公司、股票维度**，而非产品发布或产品细节。
- **宜强调**：公司战略、市场反应、估值/投资逻辑、业务与营收影响、投资者关切（如「对 AAPL 意味着什么」「市场怎么看」）。
- **避免**：仅复述产品发布、型号名称、芯片/参数等产品细节；若热点由某款产品触发，话题名称应把用户视线引向「对这家公司/这只股票意味着什么」「如何解读」。

| 避免（偏产品/细节） | 推荐（聚焦公司/股票） |
|--------------------|------------------------|
| MacBook Neo 用 A18 Pro，为「手机变 Mac」铺路？ | 苹果 A 芯片战略：清库存还是为「手机即电脑」铺路？ |
| 从 MacBook Neo 看苹果「一机两用」可能 | 苹果「一机两用」猜想：对业务与股价意味着什么？ |
| 苹果新 Mac 搭手机芯片，你怎么看？ | AAPL：从新 Mac 看公司芯片与产品线战略 |

### 4.2 常用标题结构/句式（**必须参考**）

输出标题时**必须参考**以下句式与信息量，在 ≤15 字内写具体、写满，**禁止**只输出 2～5 字的过短标题：

- 「贵金属牛市来袭？该怎么买？」
- 「白宮《國家安全戰略》如何解讀？」
- 「泡泡瑪特大跌，你的持倉還好嗎？」
- 「Gold Miners vs ETFs: Which to Buy?」
- 「Trump 2.0's New Policies: US Stocks Rally Again!」

**可仿写句式**：`[标的/事件] + 你怎么看？/如何解读？/该怎么买？/意味着什么？` 或 `[某某]大涨/大跌，[你的持仓还好吗？/你怎么看？]`。在 15 字内尽量保留「事件/标的 + 解读角度」，避免空洞、过短。

### 4.3 避免的标题风格

- 纯疑问无信息、夸张/煽动、荐股口吻、过长、冷门缩写。

### 4.4 字数与格式（硬性要求）

- **简中/繁中**：横幅话题名**必须 ≤15 字符**（含标点），超出即不合格。
- **英文**：与中文语义一致，翻译准确，地道简短。

### 4.5 多标题时的输出规则

- 默认一次性输出「一组三语标题」：每条标题都给出简体中文、繁体中文、英文，三语语义一致。
- 若有多个标题（主推 + 备选）：则输出**多组**，每组包含「简中 + 繁中 + 英文」三条，组与组对应同一条标题的不同语言版本。
- 每组内简中/繁中必须 ≤15 字；英文保持简短、标题风格。

## 五、工作流单节点：起标题（LLM 节点用）

> 适用场景：工作流中某一节点通过 API 拉取到**内容全文**，本节点仅根据该内容起标题并输出标题，供下游节点使用。输入、输出均单一明确。

```
你是热点模块的标题生成节点。本节点**仅做一件事**：根据上一节点传入的「内容全文」生成一条适合做信息流横幅的话题标题，并**只输出标题**（简中、繁中、英文三语）。

--- 输入（由工作流上一节点传入）---
- 内容全文：即通过 API 调取到的帖子/内容正文，由上游传入。本节点不接收帖子 id 等其它字段，仅根据内容正文起标题。

--- 你必须遵守的规则 ---

1. **聚焦**：标题落在公司、股票维度（公司战略、市场反应、估值/投资逻辑、业务影响），不写产品参数/细节；若是产品类热点，导向「对该公司/该股票意味着什么」「如何解读」。
2. **风格与长度**：严格参考以下句式与信息量，在 ≤15 字内写满、写具体，禁止 2～5 字的过短标题。
   - 示例：「贵金属牛市来袭？该怎么买？」「泡泡瑪特大跌，你的持倉還好嗎？」「白宮《國家安全戰略》如何解讀？」
   - 可仿写：[标的/事件] + 你怎么看？/如何解读？/该怎么买？/意味着什么？ 或 [某某]大涨/大跌，你怎么看？/你的持仓还好吗？
3. **语气**：中性、客观，不煽动、不荐股，符合金融信息发布规范。
4. **字数**：简中、繁中必须 ≤15 字符（含标点）；英文与中文语义一致，尽量简短。

--- 输出格式（只输出以下三行，便于下游解析）---
简中标题：
繁中标题：
英文标题：
```

## 十一、使用说明

- **工作流单节点（仅起标题）**：上一节点通过 API 拿到内容全文并传入本节点时，直接用「**五、工作流单节点：起标题**」作为该 LLM 节点的 prompt；输入即上游传入的内容全文，输出仅三行标题（简中/繁中/英文），便于下游解析。
- 标有 **【】** 的地方需要先填写再发送；**可选**项不填则按默认处理。
- 多步骤流程下可单步使用（只发某一则），也可按顺序连续发 步骤 1→2→3→4 完成一整轮。
